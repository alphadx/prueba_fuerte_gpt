# QA Gate Checklist (Step 1)

## Completion checks
- [ ] 100% of MVP modules documented.
- [ ] Critical flows have measurable acceptance criteria.
- [ ] User stories include Given/When/Then.
- [ ] DoD defines objective close conditions.
- [ ] Out-of-scope section present.
- [ ] No critical ambiguity remains.

## Quality checks
- [ ] Criteria are verifiable with available test data.
- [ ] Each criterion maps to evidence (DB row/event/status).
- [ ] Language avoids ambiguous terms ("rápido", "correcto", "fácil").
